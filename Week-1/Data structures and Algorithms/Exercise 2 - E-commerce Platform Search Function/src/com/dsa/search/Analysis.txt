Exercise 2: E-commerce Platform Search Function

1. Big O Notation
Big O notation is used to measure the efficiency of an algorithm by describing how its execution time or memory usage grows as the input size increases. It helps us predict worst-case performance bottlenecks before deploying code to production.

2. Complexity Analysis

Linear Search
- Best Case: O(1) (We find the target on the first try)
- Average Case: O(n) (We find it somewhere in the middle)
- Worst Case: O(n) (We have to check every single item and it's at the end, or doesn't exist)

Binary Search
- Best Case: O(1) (Target is exactly in the middle of the array)
- Average Case: O(log n) (Data space is halved repeatedly)
- Worst Case: O(log n)

3. Platform Recommendation

For an e-commerce platform with potentially millions of products, Binary Search is the clear winner. Because it cuts the search space in half with every iteration (O(log n)), it can search a massive dataset in just a few dozen steps. Linear search (O(n)) would severely degrade the user experience as the catalog grows.

The only trade-off is that Binary Search strictly requires the inventory array to be sorted first, which adds a slight overhead when adding new products, but the read/search performance gain is absolutely worth it.
